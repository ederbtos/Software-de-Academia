# Visão Geral

## Objetivo
Sistema de gestão para academias com arquitetura multi-tenant (uma base lógica por academia), cobrindo operação diária, treino, financeiro e administração.

## Problema que resolve
Centraliza processos que normalmente ficam espalhados (alunos, check-in, matrículas, cobranças, aulas, avaliações, relatórios).

## Público-alvo
- Superadmin (gestão da plataforma e academias)
- Admin de academia
- Professor
- Recepcionista

## Principais funcionalidades
- Cadastro de academias e provisionamento de schema
- Gestão de alunos e funcionários
- Matrículas, pagamentos e inadimplência
- Treinos, exercícios, avaliações físicas e aulas
- Notificações e recuperação de senha
- APIs REST e interface web server-side (Jinja)
